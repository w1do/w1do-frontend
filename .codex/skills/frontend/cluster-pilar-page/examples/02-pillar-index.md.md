## Разбор полей frontmatter

| Поле | Обяз.? | Назначение |
|------|--------|------------|
| `layout` | нет | `../../layouts/ServiceLayout.astro` — **две** `../` (pillar на 2 уровня глубже `layouts`). Фактически рендер идёт через хаб-роут, но поле оставляем для единообразия. |
| `title` | **да** | H1 и запасной `<title>`. |
| `hub: true` | **де-факто да** | Помечает услугу как хаб. Без него услуга уйдёт в `[...slug].astro` и хаб-роут может конфликтовать. |
| `seo.title` / `seo.description` | нет | Приоритетные `<title>` / meta description. |
| `description` | **да** | Короткое описание; идёт в карточку услуги на `/services`. |
| `image` | нет | Главная картинка услуги. |
| `no` | нет | Порядковый номер в общем списке `/services` (сортировка). |
| `icon` | нет | Иконка услуги в списке. |
| `tags` | нет | Метки. |
| `offers` (+`offersTitle`/`offersDescription`) | нет | Блок «Что предлагаю» — обычно 2 карточки. |
| `features` (+титулы) | нет | Блок «Преимущества» — 4 пункта (макет делит на 2+2). |
| `stats` | нет | Счётчики (value + suffix + label). |
| `faqs` | нет | Аккордеон FAQ + JSON-LD `FAQPage`. |
| `process` | нет | 4 шага (step1..step4 Title/Desc). |
| `pricing` | нет | Тарифы (items с price/features). |

## Тело (после `---`)

Интро-абзацы услуги. **Важно:** в конце тела pillar добавляй абзац с
перелинковкой на 1–2 своих spoke, например:

```md
Подробнее по направлениям: [чат-бот для Telegram](/services/ai-assistants-chatbots/telegram-chatbot)
и [чат-бот для сайта](/services/ai-assistants-chatbots/website-chatbot).
```

> Примечание про tone of voice: в исторических pillar-текстах встречается «мы» —
> это существующий контент, его намеренно не переписывали. В **новом** контенте
> (особенно в spokes) пиши строго от первого лица: «я», «меня», «мой».
