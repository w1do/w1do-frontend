# Навык: seo-urls

Универсальный навык: как из **готовой карты сайта** получить плоский список ссылок вида `домен/путь`, **по одной ссылке на строку**, — для ручного переобхода страниц в поисковых системах (в первую очередь **Яндекс.Вебмастер → Индексирование → Переобход страниц**, а также Google Search Center).

> Главный принцип: **не пиши список руками — парси его из сгенерированной карты сайта.** Карта сайта — единственный источник правды по актуальным URL (её собирает интеграция `@astrojs/sitemap`, см. навык `.junie/skills/seo-engine/SKILL.md`). Навык `seo-urls` только извлекает из карты значения `<loc>` и раскладывает их по строкам.

Триггеры навыка: «собери ссылки для переобхода Яндекса», «список ссылок домен/путь каждая с новой строки», «распарси карту сайта в список URL», «выгрузи все страницы одним списком», «seo-urls».

> **Как читать навык.** Конкретные значения (домен, каталог сборки, имя файла карты) — плейбук, а не константы. Перед работой определи фактические значения проекта (см. раздел 0) и подставляй их. В примерах: `https://<домен>` — боевой домен, `<BUILD>` — каталог сборки (`dist/` или `dist/client/`).

---

## 0. Сначала определи факты проекта

Не угадывай — собери факты:

- **Домен** — поле `site` в `astro.config.mjs`.
- **Каталог сборки** — по умолчанию `dist/`; при SSR/адаптере (`@astrojs/node` и т.п.) статика уходит в `dist/client/`. Проверь, где после `npm run build` лежат файлы карты.
- **Имя файла карты** — у `@astrojs/sitemap` это **индексный** `sitemap-index.xml`, который ссылается на `sitemap-0.xml` (и далее `sitemap-1.xml`, … при большом числе URL). В некоторых конфигурациях карта — единый файл `sitemap.xml`. Проверь, что реально лежит в каталоге сборки.
- **Формат вывода** — Яндексу нужны **абсолютные URL** (`https://домен/путь`), по одной ссылке на строку, без пустых строк и без комментариев.

> ⚠️ Список из `urls-seo.txt` — это **относительные пути** (`/about`, `/services/...`) для внутренней сверки, он **не** является готовым списком для переобхода. Для Яндекса нужны абсолютные URL из карты сайта.

---

## 1. Откуда брать карту сайта

Есть два источника, выбирай по ситуации:

1. **Локальная сборка** (рекомендуется перед деплоем) — сначала собрать проект, затем парсить файлы из каталога сборки:

```bash
npm run build
```

2. **Живой сайт** (когда нужно то, что реально отдаётся в проде) — тянуть карту по HTTP:

```bash
curl -s https://<домен>/sitemap-index.xml
```

---

## 2. Алгоритм: карта сайта → список ссылок

### Случай A. Единый файл карты (`sitemap.xml` или `sitemap-0.xml`)

Извлечь все значения `<loc>` и вывести по одной строке:

```bash
# из локальной сборки
grep -oE '<loc>[^<]+</loc>' <BUILD>/sitemap-0.xml \
  | sed -E 's~</?loc>~~g' \
  > seo-urls.txt

# или сразу из единого sitemap.xml
grep -oE '<loc>[^<]+</loc>' <BUILD>/sitemap.xml \
  | sed -E 's~</?loc>~~g' \
  > seo-urls.txt
```

### Случай B. Индексная карта (`sitemap-index.xml` со вложенными sitemap)

Индексный файл содержит **ссылки на другие sitemap-файлы**, а не на страницы. Нужно сначала собрать список вложенных карт, затем из каждой вытащить `<loc>` страниц.

Из локальной сборки (все вложенные `sitemap-*.xml` лежат рядом):

```bash
cat <BUILD>/sitemap-*.xml \
  | grep -oE '<loc>[^<]+</loc>' \
  | sed -E 's~</?loc>~~g' \
  | grep -vE '/sitemap[-0-9]*\.xml$' \
  | sort -u \
  > seo-urls.txt
```

> `grep -vE '/sitemap.*\.xml$'` отбрасывает служебные ссылки на сами sitemap-файлы (они попадают из индексного файла), оставляя только URL страниц.

С живого сайта (скачать индекс, пройтись по вложенным картам):

```bash
BASE="https://<домен>"
curl -s "$BASE/sitemap-index.xml" \
  | grep -oE '<loc>[^<]+</loc>' | sed -E 's~</?loc>~~g' \
  | while read -r sm; do curl -s "$sm"; done \
  | grep -oE '<loc>[^<]+</loc>' | sed -E 's~</?loc>~~g' \
  | sort -u \
  > seo-urls.txt
```

### Результат

Файл `seo-urls.txt` — плоский список абсолютных URL, по одному на строку:

```
https://<домен>/
https://<домен>/about
https://<домен>/contacts
https://<домен>/services
https://<домен>/services/tehpomosch
...
```

Этот список копируется/загружается в **Яндекс.Вебмастер → Индексирование → Переобход страниц**.

## 3. Правила и нюансы

- ✅ Только **абсолютные** URL (`https://домен/путь`). Относительные пути Яндекс не примет.
- ✅ **Одна ссылка — одна строка.** Без нумерации, комментариев, пустых строк в хвосте.
- ✅ Домен в ссылках должен совпадать с боевым (`site` в `astro.config.mjs`). Если в собранной карте другой домен — сначала поправь `site` и пересобери (см. `seo-engine`), не подменяй домен вручную.
- ✅ Перед выгрузкой сверь список с `urls-seo.txt` (по путям) — состав страниц должен совпадать; расхождения означают, что карта устарела (нужен `npm run build`) или `urls-seo.txt` не актуализирован (см. команду `/pages`).
- ✅ `sort -u` убирает возможные дубли и делает порядок стабильным.
- ⚠️ У **Яндекса есть суточный лимит** на количество URL для переобхода. Если ссылок больше лимита — приоритизируй: сначала новые/изменённые страницы, затем важные разделы. Файл можно разбить на части (`split -l <лимит> seo-urls.txt part_`).
- ❌ Не включай в список служебные файлы (`/sitemap*.xml`, `/robots.txt`, `/llms.txt`), картинки и `_astro/` — только страницы.
- ❌ Не редактируй список руками для «добавления» страниц: сначала создай страницу, добавь путь в `urls-seo.txt`, пересобери карту — и только потом перегенерируй список.

## 4. Чек-лист

- [ ] Определены домен, каталог сборки и имя файла карты (раздел 0).
- [ ] Карта сайта актуальна (`npm run build` выполнен, либо взята с прода).
- [ ] Список извлечён из `<loc>` карты, а не написан руками.
- [ ] В `seo-urls.txt` только абсолютные URL страниц, по одному на строку.
- [ ] Служебные `sitemap*.xml` / `robots.txt` / `llms.txt` исключены.
- [ ] Домен в ссылках = боевой домен (`site`).
- [ ] Состав URL сверен с `urls-seo.txt`.
- [ ] Учтён суточный лимит Яндекса (при необходимости список разбит на части).
