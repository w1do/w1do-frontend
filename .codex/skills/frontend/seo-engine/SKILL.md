# Навык: seo-engine

Универсальный навык: как в любом **Astro**-проекте собираются и поддерживаются два ключевых файла для поисковых систем и ИИ-роботов:

1. **Карта сайта** — `sitemap-index.xml` + `sitemap-0.xml` (генерируются автоматически при сборке).
2. **`robots.txt`** — файл правил обхода для поисковых и ИИ-ботов (поддерживается вручную).

> Главный принцип: **карту сайта генерирует интеграция, `robots.txt` ведётся вручную**. Не создавай `sitemap*.xml` руками — им управляет интеграция `@astrojs/sitemap`. `robots.txt` живёт в `public/` и всегда ссылается на актуальную карту сайта.

Триггеры навыка: «собери карту сайта», «обнови sitemap», «настрой robots.txt», «добавь бота в robots», «почему нет страницы в карте сайта», «настрой SEO-движок / индексацию».

> **Как читать этот навык.** Все конкретные значения (домен, имя проекта, каталог сборки, техпути) — это плейбук, а не константы. Перед работой **определи фактические значения из самого проекта** (см. раздел 0.1) и подставляй их. В примерах ниже используются плейсхолдеры: `https://<домен>` — боевой домен, `<PROJECT>` — имя/бренд проекта.

---

## 0. Как устроено (общая картина)

| Файл | Где лежит источник | Кто генерирует | Где итог после сборки |
| --- | --- | --- | --- |
| `sitemap-index.xml` | нет исходника (авто) | интеграция `@astrojs/sitemap` | каталог сборки (`dist/` или `dist/client/`) |
| `sitemap-0.xml` | нет исходника (авто) | интеграция `@astrojs/sitemap` | каталог сборки |
| `robots.txt` | `public/robots.txt` | **вручную**, копируется как есть | каталог сборки |
| `llms.txt`, `llms-full.txt` | нет исходника (авто) | интеграция `astro-llms-md` (если подключена) | каталог сборки |

Всё это собирается одной командой (имя скрипта смотри в `package.json`):

```bash
npm run build
```

Базовый адрес сайта задаётся **один раз** в `astro.config.mjs` через поле `site` и используется всеми интеграциями:

```js
export default defineConfig({
  site: 'https://<домен>',
  // trailingSlash по политике проекта: 'never' | 'always' | 'ignore'
  // ...
});
```

> ⚠️ Без корректного `site` карта сайта и абсолютные URL в ней сгенерируются неверно. Меняешь домен — меняй `site`, `public/robots.txt` и `name/description` в интеграциях синхронно.

### 0.1. Сначала определи фактические значения проекта

Перед любыми правками собери факты (не угадывай):

- **Домен** — поле `site` в `astro.config.mjs`.
- **Trailing slash** — поле `trailingSlash` в `astro.config.mjs` (влияет на вид URL в карте).
- **Каталог сборки** — по умолчанию `dist/`; при использовании SSR/адаптера (`@astrojs/node` и т.п.) статика уходит в `dist/client/`. Смотри `outDir`/адаптер в конфиге или просто проверь, где после `npm run build` лежат `robots.txt`/`sitemap-*.xml`.
- **Команда сборки** — раздел `scripts` в `package.json`.
- **Список актуальных URL** — если в проекте ведётся файл со списком страниц (напр. `urls-seo.txt`), используй его для сверки; если нет — сверяйся с содержимым `src/pages/**` и сгенерированной картой.

## 1. Карта сайта (sitemap) — автоматически

### 1.1. Чем генерируется

Карта собирается интеграцией [`@astrojs/sitemap`](https://docs.astro.build/en/guides/integrations-guide/sitemap/), подключённой в `astro.config.mjs`:

```js
import sitemap from '@astrojs/sitemap';

export default defineConfig({
  site: 'https://<домен>',
  integrations: [
    sitemap({
      changefreq: 'weekly',
      priority: 0.8,
      lastmod: new Date(),
    }),
  ],
});
```

Если интеграция ещё не установлена:

```bash
npx astro add sitemap
```

### 1.2. Что попадает в карту

- Интеграция обходит **все статические страницы**, которые Astro отрендерил при сборке (`src/pages/**`, включая динамические роуты вида `[slug]`, `[...slug]`, раскрытые через `getStaticPaths`).
- Итог — индексный файл `sitemap-index.xml`, который ссылается на `sitemap-0.xml` (и `sitemap-1.xml`, … при большом числе URL) со списком страниц.
- URL формируются от `site` с учётом `trailingSlash` проекта.

### 1.3. Правила и настройки

- `changefreq` — рекомендуемая частота обхода (`'weekly'` — разумный дефолт).
- `priority` — приоритет страниц по умолчанию (`0.8` — разумный дефолт).
- `lastmod` — дата последнего изменения (проставляется на момент сборки).
- Чтобы **исключить** страницу из карты, используй опцию `filter` интеграции (служебные/технические страницы). Пример:

```js
sitemap({
  changefreq: 'weekly',
  priority: 0.8,
  lastmod: new Date(),
  filter: (page) => !page.includes('/admin'),
});
```

Дополнительно доступны `customPages`, `serialize`, `i18n` и др. — см. документацию интеграции.

### 1.4. Как проверить карту

```bash
npm run build
```

После сборки проверь наличие и содержимое файлов в каталоге сборки (`dist/` или `dist/client/` — см. 0.1):

- `sitemap-index.xml` — должен ссылаться на `sitemap-0.xml`.
- `sitemap-0.xml` — должен содержать все актуальные URL проекта.

Сверь список URL в карте со списком актуальных страниц проекта (файл со списком URL, если он есть, либо `src/pages/**`).

## 2. `robots.txt` — вручную

### 2.1. Где лежит

Файл-источник: `public/robots.txt`. Всё из `public/` копируется в корень сайта как есть, поэтому он будет доступен по адресу `https://<домен>/robots.txt`.

### 2.2. Структура файла (рекомендуемый шаблон)

1. **Шапка-комментарий** с адресом сайта и (опционально) ссылкой на LLM-контекст:

```
# <PROJECT> — https://<домен>
# LLM-контекст: https://<домен>/llms.txt
```

2. **Явные разрешения для ИИ- и поисковых ботов** — каждый бот описан отдельным блоком `User-agent` + `Allow: /`. Боты сгруппированы по вендору с комментарием-заголовком:

```
# --- OpenAI ---
User-agent: GPTBot
Allow: /

User-agent: ChatGPT-User
Allow: /
```

Типичные вендорские группы: OpenAI, Anthropic, Perplexity, Google, Microsoft/Bing, Apple, Meta, Amazon, Cohere, Common Crawl, ByteDance, You.com, Diffbot, AI2, DuckDuckGo, Timpi, Omgili, Imagesift, Huawei Petal. Актуальный список ботов держи в самом файле; при необходимости уточни имена агентов в документации вендоров.

3. **Правило для всех остальных ботов** (`User-agent: *`) — разрешаем сайт, но закрываем технические/приватные пути и параметрические URL. Список `Disallow` **адаптируй под конкретный проект** (у SPA/SSR и статики он разный):

```
User-agent: *
Allow: /
Disallow: /*?
Disallow: /admin
Disallow: /api/
Disallow: /_astro/
```

4. **Ссылка на карту сайта** — обязательно в конце файла и указывает именно на индексный файл:

```
Sitemap: https://<домен>/sitemap-index.xml
```

### 2.3. Правила ведения `robots.txt`

- ✅ Добавляешь нового ИИ-бота — вставляй отдельный блок `User-agent` + `Allow: /` в соответствующую вендорскую группу (или создай новую группу с комментарием `# --- Вендор ---`).
- ✅ `Sitemap:` всегда указывает на `https://<домен>/sitemap-index.xml` (индексный файл, не `sitemap-0.xml`).
- ✅ Домен в шапке, в `Sitemap:` и в `astro.config.mjs` (`site`) должен совпадать.
- ✅ Технические/приватные пути закрываются только в блоке `User-agent: *` через `Disallow`; список подбирается под проект.
- ❌ Не переноси `robots.txt` в `src/pages` и не генерируй его скриптом без необходимости — по умолчанию он статический и лежит в `public/`.
- ❌ Не указывай в `Sitemap:` относительный путь — только абсолютный URL.

## 3. Алгоритм: собрать/обновить SEO-движок

### Шаг 1. Определить факты проекта
По разделу 0.1 узнать домен (`site`), `trailingSlash`, каталог сборки, команду сборки и источник списка URL.

### Шаг 2. Проверить/настроить карту сайта
Убедиться, что интеграция `@astrojs/sitemap` подключена в `astro.config.mjs` и `site` = боевой домен. При отсутствии — установить (`npx astro add sitemap`).

### Шаг 3. Обновить `robots.txt` (при необходимости)
- Добавить/убрать блоки ботов.
- Адаптировать `Disallow`-список технических путей под проект.
- Проверить, что `Sitemap:` ссылается на `sitemap-index.xml` с правильным доменом.

### Шаг 4. Собрать проект
```bash
npm run build
```

### Шаг 5. Проверить результат
- `robots.txt` в каталоге сборки совпадает с `public/robots.txt`.
- `sitemap-index.xml` и `sitemap-0.xml` присутствуют и содержат актуальные URL.
- Список URL в карте сайта совпадает с актуальными страницами проекта.

### Шаг 6. Актуализировать документацию
Обновить проектную документацию (напр. `SUMMARY.md`) по правилам документирования, если менялась логика индексации.

## 4. Чек-лист завершения

- [ ] `site` в `astro.config.mjs` = боевой домен.
- [ ] Интеграция `sitemap()` подключена и настроена (`changefreq`, `priority`, `lastmod`).
- [ ] `public/robots.txt` содержит нужные блоки ботов и `Disallow` техпутей проекта.
- [ ] `Sitemap:` в `robots.txt` указывает на `https://<домен>/sitemap-index.xml`.
- [ ] Домен одинаков в `astro.config.mjs`, в шапке и в `Sitemap:` `robots.txt`.
- [ ] `npm run build` проходит без ошибок.
- [ ] В каталоге сборки есть `robots.txt`, `sitemap-index.xml`, `sitemap-0.xml`.
- [ ] URL в карте сайта совпадают с актуальными страницами проекта.
- [ ] Обновлена документация (если менялась логика).
