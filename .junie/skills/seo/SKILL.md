# SEO & Контент: Правила наполнения страниц

Данный навык описывает алгоритм заполнения структуры страницы SEO-ключами, описаниями, атрибутами `alt` и методику создания полезного контента без переспама.

## 1. Подготовка и анализ семантики

Перед началом генерации контента или проектирования страницы **обязательно** выполните следующие шаги:

1.  **Сканирование папки `seo/`**: Изучите список файлов. Каждый файл или папка — это отдельный кластер запросов (например, `разработчик_ии`, `автоматизация.txt`, `LLM.txt`).
2.  **Выбор релевантного кластера**: Определите, какой файл максимально соответствует интенту создаваемой страницы.
3.  **Анализ частотности и веса**:
    - Найдите самые короткие и общие фразы в файле — это ВЧ (высокочастотные) запросы для Title и H1.
    - Найдите уточняющие фразы (с хвостами) — это СЧ/НЧ запросы для H2 и FAQ.
4.  **Планирование контента**: На основе найденных слов составьте план блоков. Каждый блок должен закрывать конкретную группу слов из кластера, чтобы не было переспама в одной секции.

## 2. Сбор семантического ядра

Используйте данные, полученные на этапе анализа:
- **H1-H2**: Должны содержать высокочастотные (ВЧ) запросы из соответствующего кластера не более 2-3 слов.
- **LSI & Синонимы**: Используйте синонимы и релевантные тематические слова (LSI), чтобы расширить семантику и избежать заспамленности основным ключом.
- **Интент**: Каждый блок должен отвечать на конкретный интент (намерение) пользователя.

## 3. Структура страницы и заголовки

### H1: Главный заголовок (Hero)
- Должен содержать основной ВЧ запрос.
- Только один на страницу.
- Пример: `Разработчик ИИ`

### H2: Подзаголовки секций
- Используются для разделения контента на логические блоки.
- Должны содержать ВЧ или СЧ запросы в естественном вхождении.
- Пример: `Услуги разработчика искусственного интеллекта`, `Стоимость разработки ИИ-решений`.

### Полезный контент (Текстовые блоки)
- **Нет воде**: Блок не должен быть "текстом ради текста".
- **Продолжение семантики**: Каждый следующий блок должен логически продолжать предыдущий, закрывая новые части семантического кластера.
- **Польза**: Текст должен нести конкретную информацию: технологии, выгоды, процессы, результаты.

## 4. Оптимизация мета-тегов и медиа

- **Title**: Основной ВЧ запрос + УТП. До 60-70 символов.
- **Description**: Краткое описание страницы с вхождением ключевых слов. До 160 символов. Призыв к действию.
- **Alt для изображений**: Описывайте, что на картинке, используя ключевые слова, но без перечисления через запятую.
    - *Плохо*: `ии, разработчик, код`
    - *Хорошо*: `Разработчик ИИ анализирует код нейросети в редакторе Cursor`

## 5. FAQ (Вопросы и ответы)

Блок FAQ является критически важным для закрытия низкочастотных (НЧ) запросов и информационного интента.
- **Количество**: Минимум 10-15 вопросов.
- **Качество**: Вопросы должны реально волновать потенциального клиента.
- **Семантика**: Используйте хвосты запросов и LSI слова в вопросах и ответах.

### Пример тем для FAQ:
1. Сроки разработки.
2. Стоимость и от чего она зависит.
3. Технологический стек (Python, LLM, RAG).
4. Безопасность данных.
5. Возможность интеграции с текущими системами.

## 6. Чек-лист проверки (Self-Audit)

1. [ ] Выполнено сканирование папки `seo/` и выбран релевантный кластер.
2. [ ] Выбран основной ВЧ запрос для Title и H1.
3. [ ] В H2 используются разные формулировки ключевых слов.
4. [ ] В тексте присутствуют LSI-слова (минимум 5-7 на блок).
5. [ ] Отсутствуют "водянистые" фразы (типа "в современном мире", "лидеры рынка").
6. [ ] В блоке FAQ не менее 10 вопросов.
7. [ ] Все изображения имеют осмысленный `alt`.
8. [ ] Текст читается легко и выглядит естественно (без "купить разработчик ии тюмень").

## 7. Использование в Astro

При передаче пропсов в компоненты, следите за тем, чтобы SEO-тексты не обрезались и корректно рендерились в HTML.

```astro
---
import Layout from '../layouts/Layout.astro';
import Hero from '../components/Hero.astro';
import Faqs from '../components/Faqs.astro';

// Данные из seo/ кластера
const title = "Разработчик ИИ: создание и обучение нейросетей";
const description = "Услуги профессионального разработчика ИИ по созданию нейросетей, интеграции LLM и автоматизации бизнес-процессов.";
---

<Layout title={title} description={description}>
  <Hero title="Профессиональный разработчик ИИ для вашего бизнеса" />
  <!-- Другие блоки -->
  <Faqs 
    title="Ответы на вопросы по разработке ИИ"
    customFaqs={[
      { question: "Сколько стоит разработка ИИ?", answer: "..." },
      // еще 14 вопросов
    ]}
  />
</Layout>
```
